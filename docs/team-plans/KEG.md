# 김은경 1차 배포 계획: 마이페이지, 커뮤니티 댓글, 팔로우/팔로잉

## 담당 범위

- 마이페이지 프로필과 활동 요약
- 보유/총 적립/총 사용 빡낌 표시
- 출석 체크 진입과 출석 상태 표시
- 내 서비스, 내 피드백, 내 게시글, 내 댓글 일부 목록과 전체 이동
- 커뮤니티 댓글 작성, 조회, 수정, 삭제
- 팔로우/팔로잉 상태, 목록, 팔로우, 언팔로우
- 대댓글은 1차 배포 범위에서 제외

## 작업 브랜치

- 기본 브랜치: `feature/mypage`, `feature/comments`, `feature/follow`
- 필요 시 세부 브랜치: `feature/mypage-summary`, `feature/follow-list`

## 프론트엔드 작업 전제

- 피그마 MCP로 생성한 프론트 기초 코드는 `frontend` 브랜치에서 먼저 구현되고, 통합 관리자가 `develop`에 병합한다.
- 김은경 작업은 `frontend`가 병합된 `develop`에서 `feature/mypage`, `feature/comments`, `feature/follow` 브랜치를 만든 뒤 시작한다.
- 마이페이지, 댓글 영역, 팔로우/팔로잉 목록과 버튼은 피그마 기반 기초 코드를 최대한 유지한다.
- 피그마 의도와 다르게 생성된 마이페이지/댓글/팔로우 화면 구조나 컴포넌트만 담당 범위 안에서 보정한다.
- 기능 연결 시 목업 프로필, 목업 활동 통계, 목업 댓글/팔로우 데이터는 실제 API 호출로 교체한다.

## 주요 계약

- 주요 테이블: `users`, `services`, `feedbacks`, `promotions`, `community_posts`, `community_comments`, `user_follows`, `point_logs`, `attendance`
- 현재 API_SPEC 관련 API: 마이페이지 API, 댓글 API, 팔로우/팔로잉 API
- API_SPEC 보완 필요: 내 게시글 목록과 내 댓글 목록 API가 없으면 추가 여부 확정

## Step 계획

### Step 1 : 기능 계약 확인

**목표**

마이페이지, 댓글, 팔로우 기능의 화면 구역과 데이터 계약을 고정한다.

**구현 기능**

- 마이페이지 프로필 영역 정의
- `frontend` 브랜치에서 생성된 마이페이지, 댓글 영역, 팔로우 컴포넌트 구조 확인
- 요약 통계 항목 정의: 내 서비스, 내 피드백, 팔로워, 팔로잉, 내 게시글, 내 댓글 수
- 댓글 응답 필드 정의: 작성자 닉네임, 댓글 내용, 작성일자
- 팔로우 상태 응답 정의
- 대댓글 제외 범위 명시

**완료 조건**

- 마이페이지에서 필요한 데이터 출처가 담당자별로 정리된다.
- 피그마 기반 마이페이지/댓글/팔로우 화면 코드에서 수정해야 할 부분이 담당 범위 안에서 정리된다.
- 댓글과 팔로우 API의 요청/응답 필드가 정리된다.
- 목업 데이터로 마이페이지와 게시글 댓글 영역이 표시된다.

**엮이는 담당자**

- 김현진: 로그인 사용자, 닉네임, 프로필
- 박재범: 빡낌과 출석 상태
- 조현정: 내 서비스
- 박건일: 내 게시글과 댓글이 붙는 게시글

**확인 방법**

- 목업 사용자로 마이페이지에 진입해 프로필, 통계, 댓글 영역이 보이는지 확인한다.

### Step 2 : 핵심 조회 흐름

**목표**

사용자가 마이페이지와 댓글, 팔로우 목록을 조회할 수 있게 한다.

**구현 기능**

- 프로필 이미지, 닉네임, 연동 계정 정보 조회
- 현재 보유 빡낌, 총 적립 빡낌, 총 사용 빡낌 조회
- 출석 여부와 연속 출석 보너스 상태 표시
- 내 서비스, 내 피드백, 내 게시글, 내 댓글 일부 목록 조회
- 팔로워/팔로잉 일부 목록과 전체 목록 이동
- 게시글별 댓글 목록 조회

**완료 조건**

- 마이페이지 요약 통계가 실제 데이터 기준으로 표시된다.
- 각 일부 목록에서 전체 목록으로 이동할 수 있다.
- 게시글 상세에서 댓글 목록을 볼 수 있다.

**엮이는 담당자**

- 박재범: 포인트와 출석 데이터
- 조현정: 내 서비스 목록
- 김현진: 내 피드백 목록
- 박건일: 내 게시글 목록

**확인 방법**

- 활동이 있는 계정과 없는 계정으로 마이페이지를 각각 조회한다.
- 게시글 상세에서 댓글 목록을 조회한다.

### Step 3 : 핵심 생성/등록 흐름

**목표**

로그인 사용자가 댓글을 작성하고 다른 사용자를 팔로우할 수 있게 한다.

**구현 기능**

- 게시글별 댓글 작성
- 댓글 작성 후 목록 즉시 반영
- 다른 사용자 팔로우
- 팔로우 후 팔로워/팔로잉 수 갱신
- 팔로우 버튼을 팔로잉 또는 언팔로우 상태로 변경

**완료 조건**

- 로그인 사용자는 게시글에 댓글을 작성할 수 있다.
- 댓글에는 닉네임, 내용, 작성일자가 표시된다.
- 팔로우 성공 후 상태와 수치가 갱신된다.

**엮이는 담당자**

- 김현진: 로그인 권한
- 박건일: 게시글 상세와 댓글 신고 대상

**확인 방법**

- 로그인 사용자로 댓글을 작성한다.
- 다른 사용자 프로필에서 팔로우를 누르고 팔로워/팔로잉 수 변화를 확인한다.

### Step 4 : 수정/삭제 또는 상태 변경 흐름

**목표**

본인 댓글과 팔로우 상태를 관리할 수 있게 한다.

**구현 기능**

- 본인 댓글 수정
- 본인 댓글 삭제
- 댓글 작성자 본인에게만 수정/삭제 버튼 표시
- 이미 팔로우한 사용자 언팔로우
- 팔로우 상태 버튼 변경

**완료 조건**

- 타인 댓글에는 수정/삭제 버튼이 보이지 않는다.
- 본인 댓글 수정 내용이 목록에 반영된다.
- 언팔로우 후 팔로워/팔로잉 수와 버튼 상태가 바뀐다.

**엮이는 담당자**

- 박건일: 댓글 신고와 블라인드 정책
- 김현진: 사용자 식별

**확인 방법**

- 본인 댓글과 타인 댓글을 각각 확인한다.
- 팔로우 후 언팔로우를 진행하고 상태 변화를 확인한다.

### Step 5 : 보상, 제한, 권한, 정책 검증

**목표**

댓글과 팔로우 권한 및 제한을 적용한다.

**구현 기능**

- 로그인 사용자만 댓글 작성 가능
- 본인 댓글만 수정/삭제 가능
- 자기 자신 팔로우 차단
- 중복 팔로우 차단
- 이미 팔로우한 사용자는 팔로우 버튼 대신 팔로잉 또는 언팔로우 버튼 표시

**완료 조건**

- 비로그인 댓글 작성이 차단된다.
- 자기 자신 팔로우와 중복 팔로우가 차단된다.
- 권한 실패 사유가 구분된다.

**엮이는 담당자**

- 김현진: 로그인과 사용자 식별
- 박건일: 댓글 신고 권한

**확인 방법**

- 비로그인 댓글 작성, 타인 댓글 수정, 자기 팔로우, 중복 팔로우를 각각 시도한다.

### Step 6 : 다른 담당 기능과의 1차 연결

**목표**

마이페이지가 다른 담당 기능의 결과를 모아 보여주게 한다.

**구현 기능**

- 내 서비스 일부 목록과 전체 목록 연결
- 내 피드백 일부 목록과 전체 목록 연결
- 내 게시글 일부 목록과 전체 목록 연결
- 내 댓글 일부 목록과 전체 목록 연결
- 보유 빡낌과 출석 체크 영역 연결
- 진행 중인 내 홍보 내역 연결

**완료 조건**

- 서비스 등록, 피드백 작성, 게시글 작성, 댓글 작성, 출석, 홍보 신청 후 마이페이지 데이터가 갱신된다.
- 마이페이지의 포인트 수치가 헤더 잔액과 일치한다.

**엮이는 담당자**

- 조현정: 내 서비스
- 김현진: 내 피드백
- 박건일: 내 게시글
- 박재범: 포인트, 출석, 홍보

**확인 방법**

- 각 활동을 1회씩 수행한 뒤 마이페이지 요약과 목록 반영을 확인한다.

### Step 7 : 에러, 빈 상태, 예외 케이스 처리

**목표**

활동이 없는 사용자와 예외 상태에서도 화면이 자연스럽게 동작하게 한다.

**구현 기능**

- 내 서비스 없음
- 내 피드백 없음
- 내 게시글 없음
- 내 댓글 없음
- 팔로워/팔로잉 없음
- 댓글 없는 게시글
- 삭제 또는 블라인드 댓글 표시 정책

**완료 조건**

- 빈 상태가 각 섹션별로 구분되어 표시된다.
- 삭제/블라인드 댓글이 일반 댓글처럼 오해되지 않는다.
- 에러 발생 시 마이페이지 전체가 깨지지 않는다.

**엮이는 담당자**

- 박건일: 블라인드 댓글 정책
- 조현정: 숨김 서비스 상태

**확인 방법**

- 신규 계정으로 마이페이지를 확인한다.
- 댓글 없는 게시글과 삭제 댓글 데이터를 확인한다.

### Step 8 : 관리자/운영 확인 또는 이력 조회

**목표**

신고와 운영 확인에 필요한 사용자 활동 정보를 조회 가능하게 한다.

**구현 기능**

- 사용자 댓글 작성 이력 조회 기준 정리
- 사용자 게시글/댓글 수 표시
- 팔로워/팔로잉 수 표시
- 신고된 댓글 작성자 식별에 필요한 필드 제공

**완료 조건**

- 댓글 신고에서 작성자와 댓글 식별자가 명확히 전달된다.
- 운영자가 신고된 댓글과 작성자를 확인할 수 있다.
- 마이페이지 요약 수치가 실제 목록 수와 맞는다.

**엮이는 담당자**

- 박건일: 댓글 신고와 관리자 신고 목록

**확인 방법**

- 댓글 신고 후 관리자 신고 목록에서 댓글 작성자와 댓글 내용을 확인한다.

### Step 9 : 통합 테스트 보강

**목표**

마이페이지, 댓글, 팔로우가 핵심 사용자 흐름에 함께 동작하는지 검증한다.

**구현 기능**

- 가입 후 마이페이지 프로필 확인
- 게시글 댓글 작성
- 댓글 수정/삭제
- 다른 사용자 팔로우/언팔로우
- 활동 후 마이페이지 요약 갱신

**완료 조건**

- `develop`에서 댓글과 팔로우 활동이 마이페이지에 반영된다.
- 마이페이지의 빡낌 잔액과 헤더 잔액이 일치한다.

**엮이는 담당자**

- 김현진, 박건일, 박재범

**확인 방법**

- 통합 테스트 계정으로 댓글 작성, 팔로우, 출석 체크, 마이페이지 재조회까지 진행한다.

### Step 10 : 배포 전 점검과 문서 정리

**목표**

마이페이지, 댓글, 팔로우 확인법을 배포 전 체크리스트로 정리한다.

**구현 기능**

- 마이페이지 표시 항목 정리
- 댓글 작성/수정/삭제 확인법 정리
- 팔로우/언팔로우 확인법 정리
- 대댓글 제외 명시
- 빈 상태 확인법 정리

**완료 조건**

- QA가 문서만 보고 마이페이지, 댓글, 팔로우 시나리오를 재현할 수 있다.
- Function_def의 김은경 담당 요구사항이 누락 없이 반영된다.

**엮이는 담당자**

- 통합 관리자

**확인 방법**

- 배포 전 체크리스트로 프로필, 통계, 목록, 댓글, 팔로우 기능을 재확인한다.
