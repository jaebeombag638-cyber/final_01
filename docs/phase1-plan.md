# BuildBack 1차 배포 PLAN

> 목표: 완성형 AI 서비스가 아니라, 빌더들이 실제로 가입하고 서비스를 등록하며 서로 피드백을 남기는 핵심 순환 구조를 빠르게 검증한다.

## 1. 1차 배포 목적

BuildBack의 장기 비전은 초보 빌더들이 서로의 첫 사용자가 되어주고, 피드백과 성장 과정을 기반으로 서비스 개선을 이어가는 공개 성장 플랫폼이다.

1차 배포에서는 AI 분석, 프로젝트 성장 일지, 팔로우, 고도화된 추천 기능보다 다음 루프가 실제로 작동하는지 검증한다.

```text
구글 로그인
    ↓
관심분야 선택
    ↓
다른 서비스 탐색
    ↓
피드백 작성
    ↓
빡낌 획득
    ↓
내 서비스 등록
    ↓
홍보 슬롯 구매
    ↓
피드백 수집
```

핵심 지표는 "가입자가 실제 피드백을 남기는가", "피드백을 남긴 사용자가 자신의 서비스를 등록하는가", "빡낌을 홍보에 사용하는가"이다.

## 2. 1차 배포 범위

### 포함 기능

- 구글 소셜 로그인
- 최초 로그인 시 관심분야 선택

- 내 서비스 등록
- 서비스 등록 조건 검증
- Google Web Risk API 기반 URL 검증
- 홍보 슬롯 구매 및 즉시 적용
- 피드백 작성 및 품질 검사
- 빡낌 지급 및 사용
- 데일리/연속 출석 미션
- 커뮤니티 게시판
- 서비스 상세 하단 유사 서비스 노출
- 서비스 신고
- 커뮤니티 글 신고

### 제외 기능

- 서비스 설명 AI 임베딩 기반 유사도 추천
- AI 피드백 작성 도우미

## 3. 사용자 정책

### 회원

회원은 구글 OAuth로만 가입한다.

회원 DB는 1차 배포 기준으로 다음 필드를 가진다.

| 필드 | 설명 |
| --- | --- |
| 회원ID | 내부 사용자 식별자 |
| OAUTH_ID | 구글 OAuth 고유 식별자 |
| 닉네임 | 서비스 내 표시 이름 |
| 포인트 | 보유 빡낌 |
| 가입일자 | 서비스 가입 시각 |

추가로 운영상 필요한 필드는 실제 구현 시 포함한다.

- 이메일
- 프로필 이미지
- 권한
- 정지 여부
- 마지막 로그인 시각

### 최초 로그인 온보딩

최초 로그인 사용자는 관심분야 선택 화면으로 이동한다.

- 관심분야 선택은 선택사항이다.
- 선택하지 않아도 서비스 이용은 가능하다.
- 선택한 관심분야는 추천 초기값으로 사용한다.
- 선택 완료 후 해당 카테고리 서비스 5개를 랜덤 추천한다.
- 해당 카테고리에 서비스가 5개 미만이면 가능한 수만 노출한다.
- 해당 카테고리에 서비스가 없으면 전체 서비스 중 랜덤 추천으로 대체한다.

## 4. 서비스 등록 정책

### 등록 조건

사용자는 아래 조건을 모두 만족해야 서비스를 등록할 수 있다.

- 가입 후 최소 24시간 경과
- 다른 서비스에 피드백 1개 이상 작성

조건을 만족하지 못한 사용자가 등록 화면에 접근하면 미충족 사유를 보여준다.

```text
아직 서비스를 등록할 수 없습니다.
- 가입 후 24시간이 지나야 합니다.
- 다른 서비스에 피드백 1개를 먼저 남겨야 합니다.
```

### 등록 정보

서비스 등록 시 다음 정보를 입력한다.

| 항목 | 필수 여부 | 설명 |
| --- | --- | --- |
| 제목 | 필수 | 서비스 이름 |
| 내용 | 필수 | 서비스 소개 및 피드백 받고 싶은 내용 |
| 카테고리 | 필수 | 관심분야/서비스 분류 |
| URL | 필수 | 실제 접속 가능한 서비스 주소 |

### 제목/내용 추천

1차 배포에서는 고도화된 AI 추천보다 입력 보조 수준으로 구현한다.

- 사용자가 URL 또는 간단 설명을 입력하면 LLM이 제목/내용 예시를 제안한다.
- 사람들의 이목을 끌면서도 서비스 내용을 잘 담고 있어야 한다.

### URL 검증

서비스 등록 전 URL을 검증한다.

- URL 형식 검증
- HTTP/HTTPS 프로토콜 허용
- 접속 가능 여부 확인
- Google Web Risk API 검사
- 위험 URL로 판단되면 등록 차단

## 6. 홍보 시스템

홍보 시스템은 빡낌을 사용해 즉시 노출을 구매하는 방식이다.

### 홍보 상품

| 상품 | 가격 | 슬롯 수 | 노출 위치 |
| --- | ---: | ---: | --- |
| 홈 화면 크게 노출 | 5000 빡낌 | 5개 | 홈 메인 영역 |
| 상단 배너 | 3000 빡낌 | 10개 | 홈/서비스 목록 상단 |
| 접속 시 팝업 | 2000 빡낌 | 제한 없음 또는 운영 설정 | 첫 접속 팝업 |

### 적용 방식

- 구매 즉시 적용한다.
- 선착순으로 슬롯을 배정한다.
- 슬롯이 가득 찬 상품은 구매할 수 없다.
- 구매 시 빡낌을 즉시 차감한다.
- 환불은 1차 배포 범위에서 제공하지 않는다.
- 동일 서비스의 중복 구매는 운영 정책에 따라 제한할 수 있다.

### 슬롯 만료

1차 배포에서는 운영 단순화를 위해 기간을 명확히 둔다.

- 홈 화면 크게 노출: 24시간
- 상단 배너: 24시간
- 접속 시 팝업: 24시간

만료된 홍보 슬롯은 자동으로 비활성화한다.

## 7. 피드백 시스템

### 피드백 작성

사용자는 다른 사람의 서비스에 피드백을 작성할 수 있다.

기본 정책:

- 본인 서비스에는 피드백 작성 불가
- 같은 서비스에 대한 중복 피드백은 1차 배포에서는 1개만 허용
- 하루 최대 3회 작성 가능
- 작성 후 한 달 이내 삭제 불가
- 수정 시에도 품질 검사를 다시 수행

### 피드백 품질 LLM 검사

피드백 등록 및 수정 시 LLM으로 품질을 검사한다.

검사 목적:

- 무의미한 내용 차단
- 욕설/비방 차단
- 복붙/스팸성 피드백 차단
- 실제 사용 경험이 부족한 피드백 차단

최소 통과 기준:

- 구체적인 장점 또는 불편한 점이 포함되어야 한다.
- 서비스와 무관한 일반 문장만 있으면 거절한다.
- 지나치게 짧은 피드백은 거절한다.

검사 실패 시 사용자가 수정할 수 있도록 사유를 짧게 보여준다.

```text
조금 더 구체적인 피드백이 필요합니다. 실제로 사용하며 좋았던 점이나 불편했던 점을 적어주세요.
```

## 8. 빡낌 지급 정책

빡낌은 사용자의 기여와 활동을 유도하는 내부 포인트이다.

### 지급 항목

| 조건 | 지급량 |
| --- | ---: |
| 최초 가입 | 500 빡낌 |
| 피드백 작성 | 200 빡낌 |
| 서비스의 첫 피드백 작성 | 추가 300 빡낌 |
| 데일리 출석 | 100 빡낌 |
| 7일 연속 출석 | 추가 200 빡낌 |
| 15일 연속 출석 | 추가 200 빡낌 |
| 30일 연속 출석 | 추가 200 빡낌 |

### 지급 제한

- 피드백 작성 보상은 일일 3회까지만 지급한다.
- 품질 검사를 통과한 피드백에만 지급한다.
- 피드백 수정은 추가 보상을 지급하지 않는다.
- 삭제 불가 기간이 끝난 뒤 삭제하더라도 지급된 빡낌은 회수하지 않는다.
- 부정 사용이 확인되면 운영자가 회수할 수 있다.

### 포인트 원장

모든 빡낌 증감은 원장에 기록한다.

| 필드 | 설명 |
| --- | --- |
| 사용자ID | 대상 회원 |
| 변경량 | 지급 또는 차감 수량 |
| 사유 | 가입, 피드백, 홍보 구매 등 |
| 관련 객체 | 서비스, 피드백, 미션 등 |
| 생성일자 | 발생 시각 |

회원 테이블의 포인트는 조회 성능을 위한 현재 잔액으로 관리하고, 실제 이력은 원장을 기준으로 추적한다.

## 11. 추천 및 탐색

### 최초 추천

최초 로그인 후 관심분야를 선택한 사용자는 해당 카테고리 서비스 5개를 랜덤으로 추천받는다.

### 유사 서비스

서비스 상세 페이지 하단에 동일 카테고리의 서비스 3개를 랜덤으로 노출한다.

- 현재 보고 있는 서비스는 제외한다.
- 동일 카테고리 서비스가 3개 미만이면 가능한 수만 노출한다.
- 1차 배포에서는 AI 유사도 계산을 사용하지 않는다.

## 16. 운영 정책

### 삭제 정책

- 피드백은 작성 후 30일 이내 삭제 불가
- 서비스 삭제는 1차 배포에서 비활성화 방식으로 처리
- 신고 누적 대상은 관리자 판단으로 숨김 처리

### 어뷰징 방지

- 동일 사용자의 일일 피드백 보상 3회 제한
- 본인 서비스 피드백 금지
- 동일 서비스 중복 피드백 금지
- URL 위험도 검사
- 커뮤니티 키워드 필터링
- 피드백 LLM 품질 검사
- 포인트 원장 기록

### 시간 기준

- 가입 후 24시간 조건: 가입 시각 기준
- 일일 제한 및 출석: KST 날짜 기준
- 홍보 슬롯 만료: 구매 시각 기준 24시간

## 17. 데이터 수집 목표

1차 배포의 진짜 목적은 완성된 기능이 아니라 향후 AI 기능을 위한 실제 데이터를 모으는 것이다.

수집해야 할 데이터:

- 사용자가 선택한 관심분야
- 서비스 제목/설명/카테고리/URL
- 피드백 본문
- 피드백 품질 검사 결과
- 서비스별 피드백 수
- 서비스 상세 조회
- 추천 서비스 클릭
- 홍보 상품 구매
- 출석 및 미션 참여
- 커뮤니티 게시글
- 신고 사유

향후 활용:

- 서비스 임베딩 기반 유사 서비스 추천
- 피드백 자동 요약
- 서비스별 개선 인사이트
- 좋은 피드백 작성 보조
- 카테고리 자동 분류
- 어뷰징 탐지

## 18. 성공 지표

### 가입/온보딩

- 구글 로그인 완료 수
- 관심분야 선택률
- 최초 추천 서비스 클릭률

### 피드백 순환

- 가입자 대비 피드백 작성률

- 피드백 작성 후 서비스 등록 전환율
- 서비스당 평균 피드백 수

### 서비스 등록

- 등록 조건 충족 사용자 수
- 서비스 등록 완료 수
- URL 검증 실패율

### 빡낌/홍보

- 빡낌 지급량
- 빡낌 사용량
- 홍보 상품 구매 수
- 홍보 슬롯 소진율

### 커뮤니티

- 게시글 작성 수
- 게시판별 작성 비중
- 신고 수

## 20. 1차 배포 완료 기준

다음 조건을 만족하면 1차 배포 가능 상태로 본다.

- 사용자가 구글 로그인으로 가입할 수 있다.
- 최초 로그인 사용자가 관심분야를 선택하거나 건너뛸 수 있다.
- 관심분야 기반 추천 서비스가 노출된다.
- 사용자가 조건 충족 후 서비스를 등록할 수 있다.
- 등록 URL이 Web Risk API로 검사된다.
- 사용자가 다른 서비스에 피드백을 작성할 수 있다.
- 피드백 작성 시 LLM 품질 검사가 실행된다.
- 피드백 보상과 첫 피드백 보상이 지급된다.
- 출석 보상이 지급된다.
- 사용자가 빡낌으로 홍보 슬롯을 구매할 수 있다.
- 홍보 구매 즉시 홈/배너/팝업 노출에 반영된다.
- 커뮤니티 게시글을 작성할 수 있다.
- 커뮤니티 글 작성 시 키워드 필터링이 동작한다.
- 서비스와 커뮤니티 글을 신고할 수 있다.
- 관리자 또는 운영자가 신고 대상과 포인트 이력을 확인할 수 있다.

## 21. 1차 배포 이후 확장 방향

1차 배포에서 수집한 실제 서비스 설명과 피드백 데이터를 기반으로 다음 기능을 추가한다.

- 임베딩 기반 유사 서비스 추천
- AI 피드백 작성 도우미
- 피드백 품질 점수 고도화
- 서비스별 피드백 인사이트 리포트
- 프로젝트 성장 일지
- 프로젝트 팔로우
- 알림
- 사용자 신뢰도 점수
- 커뮤니티 댓글과 대댓글
- 관리자 어뷰징 탐지 대시보드
