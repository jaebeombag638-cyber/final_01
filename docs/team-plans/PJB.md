# 박재범 1차 배포 계획: 홍보, 포인트 관련 기능

## 담당 범위

- 헤더 전역 빡낌 잔액 표시
- 보유 빡낌, 총 적립 빡낌, 총 사용 빡낌 계산
- 빡낌 변동 내역 최신순 페이지네이션 조회
- 출석 체크와 연속 출석 보너스
- 홍보 상품 목록/상세 조회
- 빡낌으로 홍보 신청
- 홍보 신청 모달과 미리보기
- 홍보/포인트 정책 검증

## 작업 브랜치

- 기본 브랜치: `feature/points`, `feature/promotions`
- 필요 시 세부 브랜치: `feature/attendance-rewards`, `feature/promotion-purchase`, `feature/point-header-balance`

## 프론트엔드 작업 전제

- 피그마 MCP로 생성한 프론트 기초 코드는 `frontend` 브랜치에서 먼저 구현되고, 통합 관리자가 `develop`에 병합한다.
- 박재범 작업은 `frontend`가 병합된 `develop`에서 `feature/points`, `feature/promotions` 브랜치를 만든 뒤 시작한다.
- 헤더 빡낌 잔액, 마이페이지 내 빡낌, 출석 버튼, 홍보 상품 카드, 홍보 신청 모달은 피그마 기반 기초 코드를 최대한 유지한다.
- 피그마 의도와 다르게 생성된 포인트/출석/홍보 화면 구조나 컴포넌트만 담당 범위 안에서 보정한다.
- 기능 연결 시 목업 잔액, 목업 포인트 로그, 목업 홍보 상품 데이터는 실제 API 호출로 교체한다.

## 주요 계약

- 주요 테이블: `users`, `point_logs`, `attendance`, `promotion_products`, `promotions`, `services`, `feedbacks`
- 현재 API_SPEC 관련 API: `GET /users/me/points/balance`, `POST /users/me/points/logs`, `GET /users/me/missions/attendance`, `POST /users/me/missions/attendance`, `GET /promotion-products`, `GET /promotion-products/{promotion_product_id}`, `POST /promotions`, `GET /users/me/promotions`, `GET /users/me/promotions/active`
- 핵심 정책: 빡낌 잔액 0 미만 불가, 빡낌 내역 수정/삭제 불가, 지급 빡낌 회수 없음, 홍보 환불 없음

## Step 계획

### Step 1 : 기능 계약 확인

**목표**

포인트, 출석, 홍보 기능의 금액 정책과 API 응답을 고정한다.

**구현 기능**

- 헤더 전역 잔액 표시 API 계약
- `frontend` 브랜치에서 생성된 헤더 잔액, 출석, 홍보 상품, 홍보 신청 모달 구조 확인
- 마이페이지 내 빡낌 상세 조회 계약
- 포인트 로그 최신순 페이지네이션 계약
- 출석 상태와 연속 출석 응답 계약
- 홍보 상품 목록/상세 응답 계약
- 홍보 신청 요청/응답 계약

**완료 조건**

- 포인트 잔액, 적립 합계, 사용 합계, 로그 응답 필드가 정리된다.
- 피그마 기반 포인트/출석/홍보 화면 코드에서 수정해야 할 부분이 담당 범위 안에서 정리된다.
- 홍보 상품 가격, 슬롯, 상태, 기간 선택 값이 정리된다.
- 출석 보상과 연속 출석 보너스 규칙이 정리된다.

**엮이는 담당자**

- 김현진: 가입/피드백 보상 이벤트
- 김은경: 마이페이지와 출석 버튼 위치
- 조현정: 홍보할 본인 서비스 목록

**확인 방법**

- 목업 데이터로 헤더 잔액, 마이페이지 내 빡낌, 홍보 상품 카드가 표시되는지 확인한다.

**계약 확정 문서**

- [PJB-step1-contract.md](PJB-step1-contract.md)

### Step 2 : 핵심 조회 흐름

**목표**

사용자가 빡낌 잔액, 내역, 출석 상태, 홍보 상품을 조회할 수 있게 한다.

**구현 기능**

- 현재 로그인한 사용자의 빡낌 잔액을 헤더에 전역 표시
- 마이페이지 내 빡낌 자세히 보기에서 변동 내역 최신순 페이지네이션 조회
- 보유 빡낌, 총 적립 빡낌, 총 사용 빡낌 조회
- 출석 여부와 연속 N일 출석 중 표시
- 홍보 상품 전체 목록 조회
- 홍보 상품 단건 상세 조회

**완료 조건**

- 헤더 잔액과 마이페이지 잔액이 일치한다.
- 빡낌 내역이 최신순으로 페이지네이션된다.
- 홍보 상품 카드가 신청 가능/빡낌 부족/슬롯 부족 상태로 구분된다.

**엮이는 담당자**

- 김은경: 마이페이지 내 빡낌 화면
- 조현정: 서비스 홍보 상품 노출 위치

**확인 방법**

- 로그인 후 헤더 잔액을 확인한다.
- 마이페이지 내 빡낌 자세히 보기에서 페이지를 넘겨 로그를 확인한다.
- 홍보 상품 목록과 단건 상세를 조회한다.

### Step 3 : 핵심 생성/등록 흐름

**목표**

사용자가 출석 체크를 하고 홍보를 신청할 수 있게 한다.

**구현 기능**

- 마이페이지 좌측 하단 출석 버튼
- 출석 체크 시 하루 1회 빡낌 지급
- 출석 직후 오늘 받은 빡낌 내역과 현재 잔액 토스트 표시
- 홍보 신청 버튼 클릭 시 2단 모달 표시
- 좌측 신청 폼: 상품명, 가격, 홍보할 서비스 드롭다운, 빡낌 요약
- 우측 미리보기: 서비스 미선택 안내, 서비스 선택 후 상품 노출 위치 축소 미리보기
- 홍보 신청 시 빡낌 즉시 차감 및 홍보 활성화

**완료 조건**

- 출석 성공 후 잔액과 로그가 갱신된다.
- 홍보 신청 모달에서 본인 서비스를 선택하고 신청 확인할 수 있다.
- 신청 완료 즉시 홍보가 활성화된다.

**엮이는 담당자**

- 김은경: 출석 버튼과 마이페이지 표시
- 조현정: 홍보할 서비스 드롭다운과 미리보기 데이터

**확인 방법**

- 출석 버튼을 눌러 토스트와 잔액 변화를 확인한다.
- 홍보 상품 카드에서 신청 모달을 열고 서비스를 선택해 신청한다.

### Step 4 : 수정/삭제 또는 상태 변경 흐름

**목표**

홍보 진행 상태와 만료 처리를 관리한다.

**구현 기능**

- 홍보 신청 즉시 활성화
- 홍보 기간 선택: 1일, 3일, 7일 표시
- 1차 배포에서는 1일만 활성화
- 홍보 기간 만료 시 자동 종료 처리
- 홍보 중인 서비스 라벨 표시
- 홍보 중인 서비스는 같은 상품과 다른 상품 모두 중복 신청 차단

**완료 조건**

- 활성 홍보만 홈/배너/팝업 노출 대상이 된다.
- 만료된 홍보는 종료 상태로 바뀐다.
- 이미 홍보 중인 서비스는 드롭다운에서 `홍보중` 라벨과 함께 선택 불가하다.

**엮이는 담당자**

- 조현정: 서비스 목록의 홍보 노출
- 김은경: 내 홍보 신청 내역

**확인 방법**

- 이미 홍보 중인 서비스를 다시 신청하려고 시도한다.
- 만료 시간이 지난 홍보가 종료 처리되는지 확인한다.

### Step 5 : 보상, 제한, 권한, 정책 검증

**목표**

포인트 지급/차감과 홍보 신청 정책을 엄격히 적용한다.

**구현 기능**

- 빡낌 잔액 0 미만 방지
- 빡낌 내역 수정/삭제 불가
- 피드백 삭제 시에도 지급 빡낌 회수 없음
- 하루 1회 출석 제한
- KST 기준 출석 날짜 판단
- 7/15/30일 연속 출석 달성 시 보너스 +200 지급
- 하루라도 빠지면 연속 출석 초기화
- 본인 서비스만 홍보 신청 가능
- 슬롯 소진 상품 신청 불가
- 비활성/차단 서비스 홍보 중단

**완료 조건**

- 잔액 부족 상태에서 홍보 신청이 차단된다.
- 중복 출석이 차단되고 버튼이 비활성화된다.
- 슬롯 부족 상품은 빨간색 상태로 표시되고 신청 불가하다.
- 포인트 로그는 수정/삭제되지 않는다.

**엮이는 담당자**

- 김현진: 피드백 보상과 삭제 정책
- 조현정: 서비스 소유자와 서비스 상태
- 박건일: 신고로 인한 서비스 차단

**확인 방법**

- 잔액 부족, 슬롯 부족, 중복 출석, 차단 서비스 홍보 신청을 각각 시도한다.
- 피드백 삭제 후 포인트가 회수되지 않는지 확인한다.

### Step 6 : 다른 담당 기능과의 1차 연결

**목표**

포인트와 홍보가 로그인, 피드백, 서비스, 마이페이지와 연결되게 한다.

**구현 기능**

- 최초 가입 보상 지급
- 피드백 1건당 200빡낌 지급
- 서비스 최초 피드백이면 300빡낌 지급
- 빡낌 획득/차감 시 헤더 잔액 자동 갱신
- 홍보 신청 후 서비스 노출 반영
- 마이페이지 내 홍보 신청 내역 제공

**완료 조건**

- 가입, 피드백, 출석, 홍보 신청 후 헤더 잔액이 즉시 갱신된다.
- 홍보 구매 후 서비스가 해당 노출 위치에 반영된다.
- 마이페이지에서 내 홍보 신청 내역을 확인할 수 있다.

**엮이는 담당자**

- 김현진: 가입/피드백 이벤트
- 조현정: 홍보 노출 서비스
- 김은경: 마이페이지와 헤더 잔액

**확인 방법**

- 신규 가입, 피드백 작성, 출석, 홍보 신청을 순서대로 실행하고 헤더 잔액과 로그를 확인한다.

### Step 7 : 에러, 빈 상태, 예외 케이스 처리

**목표**

포인트와 홍보 실패 사유를 사용자에게 명확히 보여준다.

**구현 기능**

- 빡낌 부족 카드 상태: 빨간색
- 슬롯 부족 카드 상태: 빨간색
- 신청 가능 카드 상태: 노란색
- 홍보할 서비스가 없을 때 안내
- 서비스 미선택 시 미리보기 안내 문구
- 이미 출석한 경우 출석 버튼 비활성화
- 로그가 없을 때 빈 상태 표시

**완료 조건**

- 홍보 카드 상태가 정책에 맞게 색상과 문구로 구분된다.
- 모달에서 서비스 선택 전후 미리보기가 다르게 표시된다.
- 실패 응답이 잔액 부족, 슬롯 부족, 권한 없음, 중복 홍보로 구분된다.

**엮이는 담당자**

- 조현정: 홍보할 서비스 없음/차단 서비스
- 김은경: 마이페이지 빈 상태

**확인 방법**

- 잔액 부족 계정, 슬롯 마감 상품, 서비스 미등록 계정으로 홍보 신청 화면을 확인한다.

### Step 8 : 관리자/운영 확인 또는 이력 조회

**목표**

운영자가 포인트와 홍보의 근거 데이터를 추적할 수 있게 한다.

**구현 기능**

- 포인트 변동 로그 저장
- 로그 필드: 사용자, 변동량, 현재 포인트, 변동 시간, 사유
- 홍보 신청 내역 저장
- 홍보 상태와 시작/종료일 저장
- 비활성/차단 서비스 홍보 중단 근거 저장

**완료 조건**

- 모든 지급과 차감은 로그로 남는다.
- 로그는 최신순 페이지네이션으로 조회된다.
- 홍보 신청과 차감 로그가 연결된다.

**엮이는 담당자**

- 김은경: 내 빡낌 자세히 보기
- 박건일: 신고로 인한 홍보 중단

**확인 방법**

- 가입, 피드백, 출석, 홍보 신청 후 각각 포인트 로그가 생성되는지 확인한다.

### Step 9 : 통합 테스트 보강

**목표**

핵심 루프에서 빡낌 획득, 사용, 홍보 노출이 끊기지 않는지 검증한다.

**구현 기능**

- 신규 가입 보상
- 피드백 보상
- 출석 보상
- 홍보 상품 조회
- 홍보 신청과 차감
- 홍보 노출 반영
- 마이페이지 내역 반영

**완료 조건**

- `develop`에서 사용자가 빡낌을 획득하고 홍보 신청에 사용할 수 있다.
- 홍보 신청 후 헤더, 마이페이지, 서비스 노출이 모두 갱신된다.

**엮이는 담당자**

- 김현진, 조현정, 김은경

**확인 방법**

- 통합 테스트 계정으로 가입, 피드백 작성, 출석, 홍보 신청, 서비스 노출 확인, 마이페이지 내역 확인을 순서대로 실행한다.

### Step 10 : 배포 전 점검과 문서 정리

**목표**

포인트와 홍보 정책을 배포 전 검증 가능한 형태로 정리한다.

**구현 기능**
